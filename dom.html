Layout

  1. Utiliza a árvore de renderização para posicionar e definir o espaço usado pelo elemento no documento
  2. Processo top > down, left > right.
  3. O elemento pai calcula sua largura
  4. Realiza iteração com o filho:
    1. Ajusta a posição (x,y) do elemento filho
    2. Chama o layout do filho, se o mesmo possui "bit incorreto". Isto calcula a altura do filho.
  5. Elemento pai calcula sua altura
  6. Elemento pai calcula seta o "bit incorreto" para falso

O que pode causar um "re-layout"

  • Mudar tamanho da fonte (ex: acessibilidade)
  • Mudar o tamanho da janela
  • Adicionar/remover estilos (com JS, por exemplo)
  • Alterações no DOM

O que fazer para evitar isso?

  • Evitar mudanças no topo da hierarquia do documento
  • Evitar estilos inline
  • Animar elementos somente se sua posição for fixed ou absoute
  • Evitar layout em tabela
