The need

  • Send hundreds of bills by snail mail
  • Mail service:
    • Provide the stamps,
    • Get the letters when they are enveloped.
  • Print shop service:
    • Get the stamps,
    • Get the virtual letters,
    • Print and envelope them,
    • Call the Mail so they can get the letters and send them to its receivers.

So how to send the bill files to the Print Shop?

  • Send hundreds of bills by snail mail
  • E-mail attachment? Not quite feasible.
  • Develop a download service on owr website? Quite feasible, but they didn't like the idea.
  • Create a WebService? That would make us develop a client to the Print Shop and it would give us more work.

Cloud storage service

Google Drive vs Dropbox

(or any other)

  • Both are reliable and fast
  • Both have version history
  • Both have reasonable price
  • Dropbox has linux client, Google Drive don't! :'(
  • Both have documentation for their APIs but Google's is huge better
  • The company was using a bunch of Google services since the beginning

Getting Started

  • Go to the Google Developers Console.
  • Create a project and set the credentials
  • Go to the Google Developers Portal, check the Google Drive APIs
  • Chose your SDK and download it.
  • Do the sample code, and check Google Drive on Github
  • Also, check my sample of Upload, Download, List and Delete files using PHP SDK. It's called Easy GD and it's also available on Github.
