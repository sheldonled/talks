Qual é o valor do Software

  • A maioria das pessoas têm dificuldade de associar valor à coisas intangíveis.
  • Se o software é uma coisa virtual, devemos pagar por ele com dinheiro real?!
  • Como as coisas virtuais são fáceis de copiar e compartilhar, deveríamos ter maneiras de protegê-las.
  • O Movimento de Software Livre tem como um dos objetivos mostrar que isso não é verdade.

Uma questão de liberdade, não de preço

0. A liberdade de executar o programa como você desejar, para qualquer propósito.

1. A liberdade de estudar como o programa funciona, e adaptá-lo às suas necessidades.

2. A liberdade de redistribuir cópias de modo que você possa ajudar ao próximo.

3. A liberdade de distribuir cópias de suas versões modificadas a outros.

E a revolução começou...

FSF -> GNU -> Linux

Debian -> DFSG -> Código Aberto -> Eric S. Raymond and Bruce Perens -> OSI -> OSD

Código aberto não simplesmente significa acesso ao código-fonte

  1. Distribuição Livre
  2. Código Fonte
  3. Trabalhos Derivados
  4. Integridade do código-fonte do autor
  5. Não discriminação contra pessoas ou grupos
  6. Não discriminação contra campos de fronteiras
  7. Distribução de licença
  8. Licença não deve ser específica para um produto
  9. Licença não deve ser restrita ao software
  10. Licença deve ser neutra tecnologicamente

Coisas desagradáveis que podem acontecer se o desenvolvedor tem o controle

  • Observar o usuário
  • Rastrear o usuário
  • Restringir o fluxo do usuário
  • Deletar dados sem o consentimento do usuário
  • Obrigar o usuário a fazer algo (ex: atualização)
  • Manter o usuário desprotegido, ao restringir da comunidade a possibilidade de melhorias de segurança
  • Vender dados do usuário

Software Livre é um negócio como qualquer outro

  • Você precisa conhecer o seu mercado
  • Quem consumirá o seu produto?
  • Qual o valor agregado ao seu produto?
  • Quem são seus competidores?
  • Ter um plano e modelo de negócios
  • Marketing
  • Ter um planejamento financeiro
  • Análise de Fluxo de Caixa
