MongoDB é uma aplicação de código aberto e de alta performance, para gerenciar e guardar informações de sua aplicação. Ele traz consigo um conceito já difundido de ser não-relacional (NoSQL) e muita facilidade no tratamento dos dados. Essa é uma apresentação introdutória, que visa mostrar de forma básica todo o universo MongoDB.

Introdução
  • NoSQL: Termo criado por Carlo Strozzi e Eric Evans
  • Não se opõe ao SQL, 'No' apenas indica que é um paradigma diferente
  • também conhecido como NoRel (Não Relacional)
Por que usar?
  • Um novo paradigma
  • Simplicidade
  • Flexível
    • Criação de bancos e "tabelas" dinâmicos
    • Adição de "colunas" em tabelas de forma dinâmica
Mais alguns motivos
  • Otimizado para escalabilidade
  • Rapidez na leitura e escrita de dados
  • Fácil aprendizado

Por que não usar

  • Não é bom com relacionamentos
  • Sem suporte nativo para transações
  • Sem a funcionalidade JOIN
Algumas características
  • Baseado em JavaScript
  • Interface (linha de comando) com sintaxe JS
  • Livre de modelagem (Schema Free)
  • GridFS - Sistema de arquivos
  • Existe versões prontas para instalação nos SO's mais comuns
  • Suporte para Humongous volume de dados
